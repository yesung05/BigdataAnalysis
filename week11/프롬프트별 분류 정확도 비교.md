# Ollama LLM HTTP 요청 분류 실험 요약

## 실험 개요

- **모델**: gemma3:4b (urlOllamahttps://ollama.com)
- **분류 대상**: HTTP 요청 100건
- **라벨 분포**: 정상 47건 / 공격 53건
- **출력 형식**: JSON (`{"label": "...", "reason": "..."}`)
- **가능한 라벨**
  - `Normal`
  - `Anomalous`

## 공통 분류 코드 구조

1. HTTP 요청을 다음과 같이 텍스트로 재구성
   ```text
   GET /example/path HTTP/1.1
   Body: ...
   ```

2. 프롬프트 템플릿에 HTTP 요청 삽입

3. `ollama.chat()` 호출
   - 모델: `gemma3:4b`
   - temperature: `0`

4. 응답에서 JSON 추출

5. 성능 평가
   - Accuracy
   - F1 Score
   - Classification Report

# 1. 기본 프롬프트 (llm_classification.pdf)

## 프롬프트 특징

가장 표준적인 영어 프롬프트.

> "You are a web security expert. Classify each HTTP request as Normal or Anomalous and provide a reason in JSON format."

### 핵심 요소

- 웹 보안 전문가 역할 부여
- Normal / Anomalous 이진 분류
- 판단 근거(reason) 생성
- JSON 형식 강제

## 단일 테스트 출력 예시

```json
{
  "label": "Anomalous",
  "reason": "SQL Injection attempt. The request includes 'OR 1=1', a common SQL injection pattern."
}
```

## 분류 성능

| 항목 | 값 |
|------|----|
| Accuracy | 0.8400 |
| F1 Score | 0.8519 |
| Unknown | 12건 |
| 총 처리 시간 | 74.7초 |
| 건당 평균 | 0.75초 |
| 1만 건 예상 | 약 124분 |

# 3. 보수적(Risk-Averse) 프롬프트 (llm_classification-Copy2.pdf)

공격 가능성이 있으면 적극적으로 `Anomalous`로 판단하도록 유도.

### 핵심 요소

- 위험 회피적 판단

# 분석 및 해석

## 가장 높은 성능

- **Copy3 (한국어 프롬프트)**
  - Accuracy: 0.8500
  - F1 Score: 0.8624

## 가장 빠른 처리 속도

- **Copy2 (Risk-Averse 프롬프트)**
  - 건당 0.45초
  - 1만 건 기준 약 75분

## 가장 낮은 성능

- **Copy1 (간결한 프롬프트)**
  - Accuracy 0.8200
  - F1 0.8269

## 주요 시사점

1. 프롬프트 설계만으로도 성능 차이가 발생한다.
2. 역할(Role)과 판단 기준을 구체적으로 제시할수록 성능이 향상된다.
3. 한국어 프롬프트도 영어 프롬프트와 동등하거나 더 좋은 결과를 낼 수 있다.
4. Risk-Averse 전략은 공격 탐지 민감도를 높이는 데 효과적이다.
